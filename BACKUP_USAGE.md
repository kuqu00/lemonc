# 备份功能使用指南

## 功能概述

Tauri 版本的财务管理系统提供了完整的本地数据备份和恢复功能：

1. **本地文件存储** - 数据保存在应用目录的 JSON 文件中
2. **全量备份** - 一键备份所有数据
3. **增量备份** - 只备份修改过的文件
4. **自动备份** - 7天自动执行全量备份
5. **备份恢复** - 从备份文件恢复数据

## 如何使用

### 1. 打开数据管理

1. 打开应用后，点击左侧菜单的 **"设置"**
2. 切换到 **"数据"** 标签页
3. 您会看到 **"数据管理（桌面版）"** 界面

### 2. 查看数据存储位置

在 **"备份管理"** 标签页中：
- 显示当前数据存储的完整路径
- 示例：`C:\Users\用户名\AppData\Roaming\com.finance.manager\` (Windows)
- 示例：`/Users/用户名/Library/Application Support/com.finance.manager/` (macOS)

### 3. 执行手动备份

#### 全量备份
- 点击 **"全量备份"** 按钮
- 备份所有数据文件
- 适合定期（如每周）执行

#### 增量备份
- 点击 **"增量备份"** 按钮
- 只备份自上次备份后修改的文件
- 速度快，适合日常使用

### 4. 查看备份列表

- 所有备份文件显示在列表中
- 显示文件名、大小和创建时间
- 备份文件按时间倒序排列

### 5. 恢复备份

1. 在备份列表中找到要恢复的备份
2. 点击该备份右侧的 **"恢复"** 按钮
3. 确认恢复对话框
4. **注意**：恢复会覆盖当前所有数据，请谨慎操作！

### 6. 删除备份

1. 在备份列表中找到要删除的备份
2. 点击垃圾桶图标
3. 确认删除

### 7. 清理旧备份

- 点击 **"清理旧备份"** 按钮
- 自动保留最新的 10 个备份
- 删除较早的备份文件

### 8. 自动备份设置

切换到 **"自动备份"** 标签页：

#### 每日首次备份
- 开启后，每天第一次打开应用时自动创建全量备份
- 确保每天都有最新的备份

#### 定时自动备份
- 开启后，按设定间隔自动创建备份
- 可设置备份间隔（5-1440 分钟）

### 9. 导入/导出

切换到 **"导入/导出"** 标签页：

#### 导出数据
- 点击 **"选择导出位置"**
- 选择保存目录
- 数据将导出为 JSON 文件

#### 导入数据
- 点击 **"选择备份文件"**
- 选择之前导出的 JSON 文件
- **警告**：导入会覆盖当前所有数据！

## 备份策略建议

### 推荐方案
1. **开启每日首次备份** - 确保每天都有全量备份
2. **每周执行一次手动全量备份** - 保留历史版本
3. **重要操作前执行增量备份** - 方便回滚

### 备份文件位置
备份文件存储在数据目录的 `backups` 文件夹中：
```
数据目录/
├── users.json          # 用户数据
├── customers.json      # 客户数据
├── loans.json          # 贷款数据
├── repayments.json     # 还款记录
├── settings.json       # 设置
└── backups/            # 备份目录
    ├── backup_full_20240101_120000.zip    # 全量备份
    └── backup_incremental_20240101_150000.zip  # 增量备份
```

## 注意事项

1. **备份文件格式**
   - 备份文件为 ZIP 格式
   - 包含所有 JSON 数据文件
   - 可直接解压查看

2. **跨平台兼容**
   - 备份文件可在不同平台间恢复
   - Windows 备份可在 macOS 恢复
   - 反之亦然

3. **数据安全**
   - 建议定期将备份文件复制到外部存储
   - 可使用云同步服务备份整个数据目录
   - 重要数据建议多重备份

4. **恢复前确认**
   - 恢复操作会覆盖当前所有数据
   - 建议在恢复前创建当前数据的备份
   - 确认恢复的目标备份是正确的版本

## 故障排除

### 备份失败
1. 检查磁盘空间是否充足
2. 检查是否有文件被其他程序占用
3. 尝试重启应用后再次备份

### 恢复失败
1. 确认备份文件未损坏
2. 检查备份文件是否为正确的格式
3. 尝试使用其他备份文件恢复

### 找不到备份文件
1. 检查数据目录路径是否正确
2. 确认 backups 文件夹是否存在
3. 检查文件权限设置
